01 // Overview
what I do
Upstream diagnosis for systems that fail under scale.

I produce written Diagnostic Artifacts that make a system’s real structure visible: what it does, what must not break, and where failure concentrates.

Most chronic complexity is not “people problems.” It is topology: misnamed primitives, unclear invariants, and enforcement living in the wrong place.

Deliverable
Diagnostic Artifact (written) — primitives, invariants, enforcement points, failure modes.
Not included
Implementation execution, delivery management, org-change programs, coaching.
Best fit
Founders, CTOs, operators, and policy leads facing recurring failure under scale.
02 // Method
what I map, in order
  1. Primitive
    01

    Name the irreducible action the system repeats.

  2. Invariants
    02

    Define the rules that must not break (fail-closed, auditable).

  3. Enforcement
    03

    Locate where rules actually live, and how violations resolve.

  4. Mismatch
    04

    Identify interface promise vs execution reality (what is recruited vs what is run).

  5. Build Map
    05

    Sequence boundaries and transitions so implementation is deterministic.
