01 // Methodwhat I map, in order
  • Primitive — name the irreducible action the system repeats.
  • Invariants — define the rules that must not break (fail-closed, auditable).
  • Enforcement — locate where rules actually live and how violations resolve.
  • Mismatch — identify interface promise vs execution reality (what is recruited vs what is run).
  • Build Map — sequence boundaries and transitions so implementation is deterministic.
